Tutorial Git

Esta página te enseñará a utilizar Git, una de las herramientas más importantes para el control de versiones.


Conceptos básicos

  • Repositorio

    Es el lugar donde se almacena tu proyecto y todo su historial de cambios.

  • Commit

    Es un punto en la historia del proyecto con cambios que has guardado.

  • Branch (Rama)

    Es una línea de trabajo independiente dentro de tu proyecto.

  • Merge

    Es la combinación de cambios de una rama a otra.
  • Clonar un repositorio

    Permite descargar una copia de un repositorio remoto a tu máquina local.

  • Crear un commit

    Después de hacer cambios, puedes crear un commit para guardar los cambios en el historial.

  • Crear una rama

    Puedes crear una rama para trabajar en nuevas funcionalidades sin afectar la rama principal.

  • Hacer un merge

    Combina el trabajo de tu rama con la rama principal o cualquier otra.

La primera sección (`git init`) sirve para inicializar un repositorio de Git en un directorio.

La segunda sección (`git add`) permite agregar archivos al área de preparación para ser incluidos en el commit.

La tercera sección (`git commit -m "mensaje"`) guarda los cambios añadidos con un mensaje que describe lo que hiciste.

La cuarta sección (`git push`) sube los commits locales al repositorio remoto, permitiendo compartir los cambios con otros.

Tipos de comandos

  • git init

  • git clone

  • git add

  • git commit

  • git push

  • Inicializa un nuevo repositorio de Git.

  • Clona un repositorio remoto en tu máquina local.

  • Añade archivos al área de preparación.

  • Guarda tus cambios en un commit.

  • Sube los commits al repositorio remoto.
